## 為什麼你這集比久方武那集難

因為你賣的是**你自己**。久方武賣的是他的觀點和框架，你賣的是你的執行力+態度。這集如果錄得保守、客氣、謙虛，會失敗。錄得自信、具體、有 edge，會成功。

**你可以說的話**：
- 「這在 2020 年是笑話，在 2026 年是日常」
- 「你已經晚了兩年，但沒關係，再晚兩年就真的晚了」
- 「AI 剛把程式碼變成水電，誰都能開」

**不要的語氣**：
- 「我也在摸索」「我不是專家」「希望大家多指教」（自貶從眾話術）
- 「現在很多人在用 AI 寫 code 啦」（稀釋自己的 edge）
- 「我只是想分享一下」（降低成交動機）

你花了一個月蓋出這個平台。你**確實**是超級個體。別客氣。

## 硬性要求

### ✅ 一定要做

1. **螢幕共享展示實物** — 不是光口述「我做了 X」，是真的打開 oxford-vision.com admin、打開抖內系統 Twitch 畫面、打開 DARKPOOL dashboard。視覺證據壓垮一切話術。
2. **講「怎麼辦到」的具體細節**——不是「我用 Claude」，是「Claude Code + Vercel 一天部署週期 + 我怎麼切 prompt 粒度讓它不亂寫」。具體度決定可信度。
3. **承認卡關和失敗**——舉 1-2 個真實踩坑（例：Next.js 16 Server Actions 的一個詭異 bug、iCloud 鬼檔讓 build 爆掉）。這讓聽眾信你是真做過。

### ❌ 一定不能

1. **不要打 prompt engineering 的販售戰**（網路免費資源太多，你輸不了也贏不了）。你的 edge 是 **end-to-end 交付**，不是 prompt 技巧。
2. **不要批評其他 AI coding 教學者**（沒必要、格局小）。你直接秀作品就夠。
3. **不要承諾「學完可以當 solopreneur 月入 X」**。你賣的是方法論和格局，不是致富保證。
